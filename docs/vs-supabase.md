# GitForms vs Supabase - Comparison

> Why choose GitForms over Supabase for form handling?

## Quick Comparison

| Feature | GitForms | Supabase |
|---------|----------|----------|
| **Price** | 100% Free | Freemium ($25/mo Pro) |
| **Setup** | 2 minutes | 15+ minutes |
| **Privacy** | Zero tracking | Analytics enabled |
| **Open Source** | 100% MIT | Partially open |
| **Self-Hosted** | GitHub only | Complex setup |
| **MCP Server** | Official | Official |
| **Learning Curve** | Minimal | Moderate |

## When to Choose GitForms

- Simple contact/lead forms
- Zero infrastructure requirement
- Privacy-first applications
- Quick prototyping
- GitHub-centric workflows
- Claude Code integration

## When to Choose Supabase

- Complex database needs
- Real-time subscriptions
- Authentication flows
- PostgreSQL features
- Large-scale applications

## Setup Comparison

### GitForms
```bash
/create-form contact name,email,message
# Done!
```

### Supabase
1. Create project
2. Set up database schema
3. Configure Row Level Security
4. Create API endpoints
5. Handle authentication
6. Deploy edge functions

## Conclusion

**GitForms**: Perfect for forms and lead capture with zero overhead.

**Supabase**: Better for full-stack applications needing database features.
